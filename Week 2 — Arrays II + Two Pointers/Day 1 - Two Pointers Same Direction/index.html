Two Pointers — Same Direction

Fast / Slow Pointer Mastery
⚡ O(n) · O(1) Space

🎯 Core Insight

Fast always moves. Slow moves only when a condition is met. Zero extra memory needed — the output is built in the same array.

if keep(arr[fast]):
  arr[slow++] = arr[fast]

⏱ 2h
slow = 1 (first element always kept; slow = next write pos)
Distance head→cycle_start = distance meeting_point→cycle_start (provable: 2S = F, 2(x+y) = x+y+nL → x = nL-y)
Negatives break monotonicity — adding elements can decrease sum, so shrinking left doesn't restore validity. Use prefix + HashMap.
Front-fill overwrites nums1 elements before reading them. Back-fill uses the empty tail — safe because we only write to positions ≥ max(i,j).
slow is the NEW LENGTH. Last valid element is at slow-1 (0-indexed).
