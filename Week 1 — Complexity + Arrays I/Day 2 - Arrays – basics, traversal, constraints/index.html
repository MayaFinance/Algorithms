Block 1: Architectural Design

The array represents the most direct abstraction of physical memory.

1. Static Arrays vs Heap Allocation

Stack (Static)

Governed by the compiler counting down the stack pointer. Nearly instantaneous allocation. Finite resource (1–8 MB).

int arr[100];

Heap (Dynamic)

Size can be resolved at runtime. Persists until `delete[]`. Higher allocation overhead and risks fragmentation without RAII.

int* arr = new int[n];

2. Address Arithmetic & O(1) Fetch

Because elements are contiguous, address calculation is a pure algebraic function. Zero-based indexing prevents $Index - 1$ calculations.

Address = Base + (Index × sizeof(Type))

C++ Pointer Translation: A dereference like `arr[i]` strictly translates to the pointer jump `*(arr + i)`. There are zero bounds checks natively.

3. std::vector & Amortization

A `vector` has `size` and `capacity`. When full, hitting `push_back(val)` forces a reallocation: an $O(n)$ process to allocate $2 \times$ space and move everything over.

Using the Potential Method ($\Phi = 2 \times Size - Capacity$), the $O(1)$ operations "save up energy" to strictly pay for the $O(n)$ reallocation moving cost, proving Amortized $O(1)$ overall.

Block 3: Constraints & Edge Cases

Checklists for Technical Interviews & CP Audits.

The Edge Case Checklist

  • ✓ Empty Arrays: Return safely (0, -1, {}, etc.) when n=0.
  • ✓ Single Element: Algorithms like 'Majority' or 'Rotations' behave uniquely at n=1.
  • ✓ Duplicates: Can Two-Sum handle [3,3] correctly for target=6 without reusing index 0 twice?
  • ✓ Negative Inputs: Are your Prefix Max/Min arrays robust against arrays consisting solely of negatives?
  • X Integer Overflow: If array elements are $10^9$ and $n=10^5$, sums exceed 32-bit `int`. Always accumulate in `long long`!

Cache Performance (Memory Wall)

Cache lines load 64-bytes at a time. An array fetch natively pre-loads the next several items into L1 Cache (Spatial Locality).

If you allocate a `matrix[R][C]`, looping C inside R (Row-major) is lightning fast. Looping R inside C (Column-major jump) causes cache misses on every access.
