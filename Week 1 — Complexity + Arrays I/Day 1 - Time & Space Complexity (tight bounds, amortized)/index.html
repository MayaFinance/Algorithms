📚 Block 1 — Formal Definitions

⏱️ 3 Hours
Big O (Upper)O(N) ≤ c·g(N)
Big Ω (Lower)Ω(N) ≥ c·g(N)
Big Θ (Tight)Strictly Linear

In mathematical terms, f(n) = O(g(n)) means there exist positive constants c and n0 such that 0 ≤ f(n) ≤ c·g(n) for all n ≥ n0.

Why drop constants? Because as N scales toward infinity, the dominant term dictates behavior. 5N^2 + 100N + 5000 is strictly bounded by c·N^2 for some massive c. The lower order terms become statistically irrelevant to CPU cycle accumulation.

Reverse engineer the solution purely from LeetCode input limits.

The "1-Second Rule" in C/C++ implies modern CPUs handle approx 108 operations per second.

  • N ≤ 10 → O(N!) [Permutations]
  • N ≤ 20 → O(2N) [Subsets]
  • N ≤ 500 → O(N3) [Floyd-Warshall]
  • N ≤ 5000 → O(N2) [Nested Loops]
  • N ≤ 105 → O(N log N) [Sorting, Binary Search]
  • N ≤ 108 → O(N) [Linear Scan]
  • N ≥ 109 → O(log N) or O(1) [Math, Binary Search on Answer]

Analyzing worst-case sequences instead of isolated events.

Amortization guarantees the average performance of each operation in the worst case.

The Dynamic Array Proof: When a vector resizes, it doubles. The cost of N insertions is:

1 + 2 + 4 + 8 + ... + N = 2N

Even though resizing is O(N), across N elements, the cost is 2N / N = 2 operations. Therefore, strictly amortized O(1).

🔥 Block 3 — Advanced Master Theorem Convergence

⏱️ 4 Hours 
for(int i=1; i<=N; i++) {
    for(int j=i; j<=N; j+=i) { ... }
}

The inner loop runs `N/1 + N/2 + N/3 + ... + N/N` times.
This factors to `N * (1 + 1/2 + 1/3 + ... + 1/N)`.
The sum of the harmonic series converges bounded by `ln(N)`. Therefore, this is precisely O(N log N). Highly common in number theory (Sieve of Eratosthenes).

If a function calls itself twice: `T(N) = 2*T(N-1)`, it produces a binary tree of height N. The number of leaves is `2^N`. The time complexity is O(2^N).
